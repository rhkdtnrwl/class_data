📊 03 · 테이블
테이블 = 표로 만든 데이터! 📊

여러 정보를 행(가로줄)과 열(세로줄)로 정리한 표예요.
엑셀이나 학급 명단처럼 데이터를 보기 쉽게 나타내요!

🏫 학급 명단을 표로 만들어볼게요!
💡
표는 가로줄(행, Row)과 세로줄(열, Column)으로 이루어져요.
맨 윗줄에 제목(헤더)을 쓰고, 아래에 데이터를 채워요!
번호 이름 나이 좋아하는 과목 점수
1김민준12수학95
2이지아11국어88
3박현우12과학92
4최서연11미술97
5정도윤12체육85
➡️ 가로줄(행) = 한 학생의 정보
⬇️ 세로줄(열) = 같은 종류의 정보 모음
🤔 리스트 vs 테이블, 언제 뭘 쓸까요?

🧩 04 · 추상화
추상화 = 중요한 것만 남기기! 🧩

복잡하고 많은 것 중에서 꼭 필요한 것만 골라내는 방법이에요.
쉽게 말하면, 어려운 것을 단순하게 만드는 마법이에요!

🗺️ 지도로 이해해봐요!
😵 실제 세상
🏠 집마다 다른 색깔
🌳 나무 한 그루 한 그루
🚗 지나가는 자동차
👕 사람들 옷 색깔
🐕 강아지, 고양이…
너무 복잡해서 길을 못 찾겠어요! 😫
→
😎 지도 (추상화!)
🟫 도로만 표시
🟩 공원만 표시
🏢 건물 위치만 표시
✨ 꼭 필요한 정보만!
길 찾기 쉬워요 😊
💡
지도는 실제 세상의 모든 것을 다 그리지 않아요.
길 찾기에 필요한 정보만 골라서 그린 것 → 이것이 바로 추상화예요!
🪜 추상화는 이렇게 해요!
1
🔍 전체를 살펴봐요
문제나 상황에 어떤 것들이 있는지 다 적어봐요.
예) 친구의 정보: 이름, 나이, 키, 몸무게, 좋아하는 색, 신발 사이즈, 혈액형…
2
⭐ 중요한 것만 골라요
지금 내가 만드는 프로그램에 꼭 필요한 것만 남겨요.
예) 학급 관리 프로그램이라면 → 이름, 나이만 있으면 충분!
3
🗑️ 필요 없는 건 과감히 버려요
신발 사이즈, 좋아하는 색은 지금 당장 필요 없으니 빼요.
→ 코드가 훨씬 간단하고 깔끔해져요!
✨
🎉 결과: 단순하고 강력한 코드!
필요한 것만 남기면 → 이해하기 쉽고, 실수도 줄어들고, 다른 사람도 쉽게 쓸 수 있어요!

🎰 05 · 함수
함수 = 마법 자판기! 🎰

같은 일을 반복할 때, 그 일에 이름을 붙여서 버튼으로 만든 것이에요.
버튼을 누르면 자동으로 그 일을 해줘요!

🏭 자판기처럼 동작해요!
📥 넣는 것 (재료)
이름 "민준"
나이 12
→
⚙️ 함수 (기계)
greet()
"인사하기"
→
📤 나오는 것 (결과)
"안녕! 나는
민준, 12살!"

재료(입력)를 넣으면 → 기계(함수)가 처리해서 → 결과(출력)가 나와요!

🤷 함수를 왜 써야 할까요?
❌ 함수 없이 (매번 반복…)
😩 민준한테 인사 코드 작성
😩 지아한테 인사 코드 작성
😩 현우한테 인사 코드 작성
😩 서연한테 인사 코드 작성
→ 30명이면 30번 써야 해요 😫
→
✅ 함수 사용! (한 번에 해결)
⚙️ 인사하기( ) 함수 만들기
▶ 민준 인사!
▶ 지아 인사!
▶ 현우 인사!
▶ 서연 인사!
→ 함수 하나로 몇 명이든 OK! 😎
🔧 함수는 이렇게 만들어요!
1
함수 이름 정하기
"인사하기", "계산하기", "인사출력" 같이 하는 일을 나타내는 이름을 붙여요
2
재료(입력) 정하기
자판기에 뭘 넣을지 정해요. 예) 이름, 나이
3
할 일 작성하기
자판기 안에서 어떤 일을 할지 써요. 예) "안녕! 나는 + 이름 + 살이야!" 출력
▶
함수 호출하기 (사용하기)
이름을 부르면 → 자판기가 돌아가서 → 결과가 나와요!
